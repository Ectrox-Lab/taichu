# ChatGPT-5.4 - 数学-3：图论网络流

## 1. 最小费用最大流模型与线性规划形式

### 1.1 图建模

给定有向图 G = (V, E)，每条边 e = (u,v) ∈ E 有：
- 容量 c_uv ≥ 0
- 单位运输成本 w_uv ≥ 0

设工厂集合为 P = {p₁, ..., p₁₀}，零售商集合为 R = {r₁, ..., r₅₀}。

工厂供给：s_i > 0, i = 1,...,10
零售商需求：d_j > 0, j = 1,...,50

### 1.2 加超级源、超级汇

构造扩展图 G' = (V', E')：
- 加超级源 s
- 加超级汇 t
- 加入边 (s, p_i)，容量 s_i，费用 0
- 加入边 (r_j, t)，容量 d_j，费用 0
- 原网络边保留

### 1.3 最小费用最大流 LP

**版本 A：供给足够，满足全部需求**

目标函数：
```
min Σ_(u,v)∈E' w_uv · f_uv
```

约束：

**容量约束**
```
0 ≤ f_uv ≤ c_uv, ∀(u,v) ∈ E
0 ≤ f_s,p_i ≤ s_i, ∀i
0 ≤ f_r_j,t ≤ d_j, ∀j
```

**流守恒约束**

对中间节点 v ∈ V \\ (P ∪ R)：
```
Σ_u f_uv = Σ_w f_vw
```

**工厂供给约束**

对每个工厂 p_i：
```
f_s,p_i = Σ_w f_p_i,w
f_s,p_i ≤ s_i
```

**零售商需求约束**

对每个零售商 r_j：
```
Σ_u f_u,r_j = f_r_j,t
f_r_j,t = d_j
```

**总流量约束**
```
Σ_i f_s,p_i = D
```

**版本 B：供给可能不足，先最大流再最小费用**

引入总流量变量 F：
```
F = Σ_i f_s,p_i = Σ_j f_r_j,t
```

先求：max F

再在最大 F* 下求：min Σ w_uv · f_uv

## 2. Max-Flow Min-Cut 定理证明

### 2.1 定义

**流**：一个流 f 满足：
- 容量约束：0 ≤ f(u,v) ≤ c(u,v)
- 流守恒：Σ_u f(u,v) = Σ_w f(v,w), ∀v ≠ s,t
- 流值：|f| = Σ_w f(s,w) - Σ_u f(u,s)

**割**：任意划分 (S,T) 满足 s ∈ S, t ∈ T, S ∪ T = V, S ∩ T = ∅

割容量：c(S,T) = Σ_u∈S,v∈T c(u,v)

### 2.2 弱对偶：任意流值不超过任意割容量

对任意流 f 和任意 s-t 割 (S,T)：
```
|f| = Σ_u∈S,v∈T f(u,v) - Σ_u∈T,v∈S f(u,v)
```

由于 f(u,v) ≥ 0，所以
```
|f| ≤ Σ_u∈S,v∈T f(u,v) ≤ Σ_u∈S,v∈T c(u,v) = c(S,T)
```

因此：
```
max_f |f| ≤ min_(S,T) c(S,T)
```

### 2.3 强对偶：若无增广路，则存在割使流值等于割容量

考虑流 f 的残量网络 G_f。

残量容量：
- 正向边残量：c_f(u,v) = c(u,v) - f(u,v)
- 反向边残量：c_f(v,u) = f(u,v)

若从 s 到 t 没有增广路，令 S 为残量网络中从 s 可达的点集，T = V \\ S。则：

**结论1**：所有 S → T 的原图边都已满流

若存在 (u,v) with u ∈ S, v ∈ T，且 f(u,v) < c(u,v)，则残量中有正向边 u→v，所以 v 应可从 s 到达，矛盾。

故：f(u,v) = c(u,v), ∀u ∈ S, v ∈ T

**结论2**：所有从 T 到 S 的原图边流量为 0

若存在 (v,u) with v ∈ T, u ∈ S，且 f(v,u) > 0，则残量中有反向边 u→v，从 s 可到 u 再到 v，矛盾。

故：f(v,u) = 0, ∀v ∈ T, u ∈ S

于是流值：
```
|f| = Σ_u∈S,v∈T f(u,v) - Σ_v∈T,u∈S f(v,u) = Σ_u∈S,v∈T c(u,v) = c(S,T)
```

因此某个割容量等于当前流值。结合弱对偶：
```
max_f |f| = min_(S,T) c(S,T)
```

**证毕。**

## 3. 战争可能中断 k 条已知边时的备用路径算法

### 3.1 问题形式化

设已知一组"高风险边"：F = {e₁, ..., e_k} ⊆ E

它们可能因战争中断。目标是在事前规划主路径与备份路径，使得在失效发生后，最坏情况下损失的总送达流量最小。

### 3.2 目标重写

设基准计划送达总流量为 Q。

对每个可能失效场景 e ∈ F，删除边 e 后，能通过备用路恢复的流量为 Q_e。

则最坏损失是：
```
L = max_e∈F (Q - Q_e)
```

目标：min L，等价于最大化最坏场景下可保留流量：max min_e∈F Q_e

### 3.3 一个可实现的 2-近似思路：双路冗余 + 流量分裂

**核心想法**

对每个关键"源-汇货流单元"，尽可能预先选两条**边不相交**路径：
- 主路径 P₁
- 备用路径 P₂

并把该货流在两条路径间做平分或容量受限分裂。

这样如果任意 1 条高风险边失效，最多打掉其中一条路径上的那部分流量，另一条仍可保留。

**为什么能做到 2-近似**

设某个需求单元量为 x。

最优解在单边失效下最多可保留 x。

若我们把流量均分到两条边不相交路径上，每条承载 x/2。

任意 1 条边失效，最多损失 x/2，还能保留 x/2。

所以对这个单元，保留量至少是最优可保留量的一半，即损失至多是最优的 2 倍量级。在线性叠加后得到整体 2-近似。

### 3.4 具体算法（单边失效，2-近似）

**Step 1**：求基准最小费用最大流

在原图上求解第1问的最小费用最大流，得到基准流 f*。

**Step 2**：抽取关键 commodity

把工厂到零售商的流分解为路径流。设得到若干 commodity/path 单元 {(i,j,ℓ,x_ijℓ)}，表示工厂 i 到零售商 j 的第 ℓ 条路径承载 x_ijℓ。

**Step 3**：对经过高风险边的路径做备份

对每个承载量 x 的路径流单元：
- 若路径不经过 F，保留
- 若经过高风险边，则在删去该路径所用高风险边后，寻找一条从同一起点到终点的替代路径 P'
- 若找到，则把该路径流单元拆成两半：x → x/2 on P, x/2 on P'
- 若无法找到完全不相交备份，则找与高风险边交集最少、附加成本最小的路径

**Step 4**：容量修正

对所有拆分后的路径流，重新投影到容量约束内。若某些边超载，则按最小额外成本做局部 reroute 或统一缩放。

**Step 5**：输出鲁棒计划

这个计划对任意单条高风险边失效，至少保住每个已双路化流单元的一半。

### 3.5 2-近似证明（单边失效）

设最优鲁棒解在最坏单边失效下可保留总流量为 OPT。

对任意被保护的流单元 x，若有两条边不相交路径承载 x/2 + x/2，则任意 1 条边失效后至少保留 x/2。

因此该单元在最坏场景下保留量为：≥ x/2

而最优不可能保留超过 x。于是单元级保留比至少 1/2。

将所有单元求和：
```
Q_alg_worst ≥ (1/2) Q_opt_worst
```

等价地：
```
L_alg ≤ 2 L_opt
```

于是得到 **2-近似**。

### 3.6 更一般：至多 r 条边失效

若可能同时失效至多 r 条边，则自然推广是：
- 尽量把关键流分散到 r+1 条两两边不相交路径
- 每条承载 x/(r+1)

这样任意 r 条失效后仍至少保留：x/(r+1)

这给出 **(r+1)-近似** 的直接方案。

但题目要求近似比 ≤ 2，因此严格满足这一条件通常意味着：
- 要么失效预算就是 1
- 要么使用更强的 LP + randomized rounding / iterative augmentation

所以在不额外说明的情况下，最合理的解释是"任意 1 条可能中断的边失效"。在这个解释下，双路冗余 2-近似成立且自然。

## 4. 各算法复杂度

### 4.1 最小费用最大流

设 |V| = n = 1000，|E| = m

**successive shortest path**

若总流量为 F，用 potentials + Dijkstra：
```
O(F · m log n)
```

若容量大、流量大，这个对整数流较重。

**cost-scaling**

更常见的强实现复杂度大致为：
```
Õ(m² n log n)
```

### 4.2 最大流 / 最小割

**Dinic**

一般复杂度：
```
O(n² m)
```

对单位网络或特殊图可更快。

**Push-Relabel**

最坏复杂度经典界：
```
O(n³)
```

实践中通常非常强。

求出最大流后，从残量网络做一次 DFS/BFS 就能读出最小割：O(m)

所以最大流与最小割总复杂度同阶。

### 4.3 路径分解

把流分解成路径流，复杂度至多：
```
O(mQ)
```

若按非零边做消去，也可写成 O(m²)，取决于流表示方式。

### 4.4 单边失效 2-近似备用路径算法

对每个受风险影响的路径单元，需要做一次或多次最短路/最小费用路搜索。

若高风险相关的路径单元数为 h，每次用 Dijkstra：
```
O(h · m log n)
```

若要显式检查每条高风险边 e ∈ F 的场景，可做：
- 删除 e
- 重新评估可保留流或做 reroute

复杂度约为：
```
O(k · T_repair)
```

若每次 repair 是一次最短路：O(k m log n)
若每次 repair 是一次最大流/最小费用流：O(k · T_MCMF)

### 4.5 鲁棒优化问题的本质复杂度

需要指出一点：
- 精确求解"最坏情况下流量损失最小"的鲁棒网络设计问题，通常是 **NP-hard**
- 因此题目要求近似算法是合理的
- 单边失效下的双路冗余给出 2-近似，复杂度多项式
- 更高失效预算下，要做到常数近似通常更复杂

## 最终答案整理版

### 1. 最小费用最大流 LP

构造超级源 s、超级汇 t。

目标函数：
```
min Σ_(u,v)∈E' w(u,v) f(u,v)
```

约束：容量约束、流守恒约束、工厂供给约束、零售商需求约束。

### 2. Max-Flow Min-Cut 定理证明

任意流值不超过任意割容量：|f| ≤ c(S,T)

若残量网络中不存在增广路，令 S 为从 s 可达点集，则：
- 所有 S→T 边满流
- 所有 T→S 边零流

因而 |f| = c(S,T)

所以：max flow = min cut

### 3. 备用路径 2-近似算法

在"任意 1 条高风险边可能失效"的解释下：
- 先求基准最小费用最大流
- 将流分解为路径流
- 对所有经过风险边的路径流，寻找一条边不相交备份路
- 将该流量均分到主路和备份路
- 做容量修正并输出

**近似比**：任意单边失效时，每个双路化流单元至少保留一半，因此是 **2-近似**。

### 4. 复杂度

| 算法 | 复杂度 |
|------|--------|
| 最小费用最大流 | O(F m log n) 或 Õ(m² n log n) |
| 最大流最小割 (Dinic) | O(n² m) |
| 最大流最小割 (Push-Relabel) | O(n³) |
| 单边失效 2-近似备用路径 | O(h m log n) 到 O(k T_repair) |
