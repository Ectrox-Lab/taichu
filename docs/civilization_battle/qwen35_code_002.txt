## 3. 代价模型

### 基本代价因子

```python
class CostModel:
    def __init__(self):
        self.seq_page_cost = 1.0
        self.random_page_cost = 4.0
        self.cpu_tuple_cost = 0.01
        self.cpu_index_tuple_cost = 0.005
```

### 各操作代价

**SeqScan**：
```
Cost = pages × seq_page_cost + rows × cpu_tuple_cost
```

**IndexScan**：
```
Cost = index_pages × random_page_cost + rows × (random_page_cost + cpu_tuple_cost)
```

**NestedLoopJoin**：
```
Cost = N_outer × cost_inner + N_outer × N_inner × cpu_operator_cost
```

**HashJoin**（假设内存足够）：
```
Cost = cost_left + cost_right + (|left| + |right|) × cpu_operator_cost
```

**Sort**：
```
Cost = 2 × data_size × seq_page_cost + rows × log(rows) × cpu_operator_cost
```

## 4. 计划枚举（动态规划）

```python
class Optimizer:
    def optimize(self, query):
        tables = list(query.tables)
        best = {}  # subset -> PlanNode
        
        # 初始化单表
        for table in tables:
            plan = self.create_scan_plan(table, query.conditions.get(table))
            best[frozenset([table])] = plan
        
        # 动态规划枚举连接
        for size in range(2, len(tables) + 1):
            for subset in combinations(tables, size):
                key = frozenset(subset)
                best_plan = None
                
                # 枚举所有分割
                for left in proper_subsets(subset):
                    right = subset - left
                    left_plan = best[frozenset(left)]
                    right_plan = best[frozenset(right)]
                    
                    # 尝试不同连接方法
                    for method in ['nested_loop', 'hash', 'merge']:
                        if self.can_apply(method, left_plan, right_plan):
                            cost = self.cost_model.join_cost(method, left_plan, right_plan)
                            if best_plan is None or cost < best_plan.cost:
                                best_plan = JoinNode(left_plan, right_plan, method, cost)
                
                best[key] = best_plan
        
        # 添加聚合
        final_plan = self.add_aggregation(best[frozenset(tables)], query.group_by)
        return final_plan
```

## 5. 物化策略

**决策依据**：
- 中间结果需要多次使用 → 物化
- 下游操作需要随机访问 → 物化
- 中间结果很小 → 可物化
- 内存足够 → 流水线

**HashJoin**：必须物化内表构建哈希表
**NestedLoopJoin**：可流水线驱动外表
**Sort**：必须物化（外排序）

```python
def should_materialize(self, plan, parent_requirements):
    if plan.output_rows < MEMORY_THRESHOLD:
        return False  # 小结果，可直接传递
    if parent_requirements.random_access:
        return True  # 需要随机访问，物化
    if plan.operator in [SORT, HASH_AGG]:
        return True  # 操作本身需要物化
    return False  # 默认流水线
```
