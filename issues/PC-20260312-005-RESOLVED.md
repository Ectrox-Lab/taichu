### Expert Rationale (82 words)

Original proposal technically sound but over-engineered for Phase A scale. HSM and blockchain add significant cost and complexity without proportional security benefit at current volume. Recommend simplified approach: cryptographic signatures using software-backed keys provide tamper-evident audit trail at fraction of cost. HSM migration justified when governance volume exceeds 50 decisions/month. Blockchain anchoring should be evaluated as separate strategic_initiative with clear ROI. Current simplified scope maintains security properties while remaining operable.

### Conditions (Expert Mandated)

1. **Key Management**
   - Use AWS KMS or equivalent (not self-managed)
   - Key rotation every 90 days
   - Multi-person authorization for key access

2. **Signature Scope**
   - APPROVE/REJECT decisions only (not REVISE/DEFER)
   - Signature includes decision hash + timestamp
   - Verification endpoint publicly accessible

3. **Audit Requirements**
   - All signatures logged to append-only store
   - 7-year retention
   - Quarterly signature verification audit

4. **Future HSM Migration**
   - Design must support HSM cutover
   - Budget request for Phase C if volume >50/month
